我的 VS Code 基本設定

2025-07-18

本文記錄我的 VS Code 基本設定。

Version

VS Code 1.102.1

VS Code

$ brew install --cask visual-studio-code
  • 使用 Homebrew 安裝 VS Code

Theme

ext install eenaree.webstorm-new-dark
  • WebStorm 配色主題

GitHub Copilot

ext install github.copilot
  • 使用 AI 幫助

Key Bindings

ext install k--kato.intellij-idea-keybindings
  • 沿用 WebStorm 的 Shortcut

keybindings.json

[
  {
    "key": "cmd+s",
    "command": "workbench.action.files.save",
    "when": "editorTextFocus"
  }
]
  • ⌘ + S：存檔

WebStorm New UI Theme 預設並沒有 ⌘ + S，要手動補上

User Settings

  • 全部 VS Code 客製化設定

Window

"window.zoomLevel": 1.6
  • 調整整體 UI 的縮放層級（不是字體大小），1.6 表示放大 160%。適合高解析螢幕或眼睛疲勞時使用
"window.commandCenter": false,
  • 關閉標題列上的 Command Center 輸入框

Workbench

"workbench.colorTheme": "Webstorm New Darcula"
  • 使用名為 Webstorm New Darcula 的主題
"workbench.sideBar.location": "right"
  • 將側邊欄（如 Explorer）移至右側

預設是左側

"workbench.editor.labelFormat": "short"
  • Tab 上顯示簡短檔名

只顯示檔名，不含路徑

"workbench.editor.untitled.labelFormat": "name"
  • 對未命名檔案（Untitled）只顯示名稱，不顯示路徑或編號
"workbench.activityBar.location": "hidden"
  • 隱藏 Activity Bar

左邊欄的圖示欄，如資源管理器、Git、擴充功能等

"workbench.editor.showTabs": "none"
  • 隱藏分頁列 (每次只會顯示單一檔案的標題)
"workbench.statusBar.visible": false
  • 隱藏 VS Code 視窗底部的 Status Bar
"workbench.tree.enableStickyScroll": false
  • Explorer 不使用 sticky scroll
"workbench.layoutControl.enabled": false
  • 隱藏 VS Code 右上角的版面配置控制器 (Layout Control)
"workbench.secondarySideBar.defaultVisibility": "hidden"
  • 藏次側邊欄 (需手動開啟)
"workbench.colorCustomizations": {
  "statusBar.background": "#303030",
  "statusBar.noFolderBackground": "#303030",
  "focusBorder": "#303030"
}

自訂工作區顏色：

  • statusBar.background：開啟資料夾時底部狀態列的背景色
  • statusBar.noFolderBackground：未開資料夾時底部狀態列的背景色
  • focusBorder：焦點（選取）元件的外框顏色，例如表單欄位邊框

Editor

"editor.fontSize": 15
  • 編輯器文字大小設定為 15px
"editor.fontWeight": "500"
  • 編輯器字體粗細為 500

略粗於正常 400，但不如 bold

"editor.fontFamily": "Fira Code"
  • 使用 Fira Code 字型

支援程式語言常用連字，如 !== → ≠

"editor.fontLigatures": true
  • 啟用字型連字

如 =>、=== 顯示為合併符號

"editor.lineHeight": 1.8
  • 設定行高為 1.8 倍字體大小
"editor.tabSize": 2
  • Tab 縮排寬度為 2 個空白字元
"editor.minimap.enabled": false
  • 關閉 minimap

右側的小型預覽地圖

"editor.stickyScroll.enabled": false
  • 編輯器不使用 sticky scroll
"editor.renderWhitespace": "all"

讓 VS Code 在編輯器中顯示所有空白字元，包含：

  • 空格 (space)：顯示小點
  • Tab 字元：顯示箭頭 →
  • 行尾多餘空白：顯示小點
  • 單詞間的空格：顯示小點
"editor.renderControlCharacters": true

設定控制 VS Code 是否在編輯器中顯示控制字元 (Control Characters)：

  • 控制字元指的是不可見字元，例如：
  • NULL (\0)
  • 換頁符 (Form Feed \f)
  • 換行符號以外的控制碼
  • 其他非列印 ASCII 碼
"editor.scrollbar.horizontal": "hidden"
"editor.scrollbar.vertical": "hidden"
  • 編輯器不顯示水平與垂直卷軸
"editor.parameterHints.enabled": false
  • 關閉函式參數提示

當輸入函式時不會自動跳出參數資訊

"editor.accessibilitySupport": "off"
  • 關閉無障礙輔助功能支援

例如讀屏功能，可微幅提升效能

"editor.tabCompletion": "on"
  • 啟用 tab 鍵補全功能

在輸入時按下 Tab 可補全片段或建議

"editor.acceptSuggestionOnEnter": "off"
  • 按下 Enter 不會自動接受 autocomplete 提示，避免誤選
"editor.linkedEditing": true,
  • 自動同步修改 HTML 前後 tag
"editor.cursorBlinking": "smooth",
  • 設定游標為 平滑淡入淡出 閃爍

Explorer

"files.exclude": {
  "**/.idea": true,
  "**/.vscode": true,
  "**/.git": true,
  "**/.DS_Store": true,
  "**/Thumbs.db": true,
  "**/node_modules": true,
  "**/public": true,
  "**/resources": true,
  "*.lock": true
}
  • 設定 Explorer 不顯示 的 目錄 與 檔案

Git

"git.openRepositoryInParentFolders": "never"
  • 不會自動搜尋並打開上層資料夾中的 Git repository

這可避免誤開大型 monorepo

Conclusion

  • VS Code 可設定的地方非常多，且都可以 JSON 格式保存
